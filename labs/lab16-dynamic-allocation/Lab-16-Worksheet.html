Lab 16: Dynamic Allocation & Arrays

Introduction to Computer Programming (ICP)
Academic Year: 2025-2026
Overview: This lab covers manual memory management in C++. You will practice allocating memory on the Heap using new and new[], and ensuring proper cleanup with delete and delete[].

Exercise 1: Dangling Pointers Lecture 16, Slide 32

Learning Objective: Understand Undefined Behavior (UB) and Pointer Safety.
Create a program that allocates an integer, deletes it, and then attempts to access its value. Use the provided template ex1_dangling.cpp.

Requirement: After observing the behavior, modify your code to set the pointer to nullptr immediately after deletion. 
int* ptr = new int(42);
delete ptr;
// access ptr here -> UB!
ptr = nullptr; // Safety first

Exercise 2: Memory Leak Detection Lecture 16, Slide 43

Learning Objective: Track resource allocation vs. deallocation.
Implement a manual counter to track memory blocks. In ex2_leak.cpp, simulate a mistake where you allocate two integers but only free one.

Task: Verify that your counter remains non-zero at the end of the program, indicating a leak. Then, fix the leak.
Pro Tip: In professional environments, tools like Valgrind automate this process.

Exercise 3: Dynamic String Arrays Lecture 16, Slide 68

Learning Objective: Handle variable-length data entered by the user.
Write a program (ex3_strings.cpp) that asks the user for a number n and then allocates a dynamic array of exactly n strings.

Task: Read strings into the array and print them back to the console with their indices (e.g., [0]: Hello).

Exercise 4: 2D Dynamic Matrices Lecture 16, Slides 63-67

Learning Objective: Master pointer-to-pointer (int**) structures.
Manually build a 2D matrix in ex4_matrix.cpp. You must:
  1. Allocate an array of row-pointers.
  2. For each pointer, allocate a column array.
  3. Initialize m[i][j] = i + j.
  4. Deallocate in reverse order.

Exercise 5: The "Grow" Operation Lecture 16, Slides 58-59

Learning Objective: Simulate how std::vector resizes internally.
Complete the grow() function in ex5_growth.cpp. When the array becomes full, your code should:
  • Allocate a new array with 2x the current capacity.
  • Copy all existing data.
  • Delete the old memory.
  • Return the new pointer.
